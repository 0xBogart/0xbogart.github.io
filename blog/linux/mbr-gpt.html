CREAR PARTICIONES

MBR & GPT




Una tarea bastante común como Administrador de Sistemas es lo que se conoce como Storage Management o Administración del almacenamiento.

Tareas como agregar/formatear discos/particiones {virtuales o físicos}, liberar espacio, ampliar file systems y puntos de montaje, entre otras.

Principalmente se distinguen 2 tipos de particiones de disco, MBR y GPT.


☣ MBR {MASTER BOOT RECORD}

Esta tabla de particiones data de 1983 y se introdujo de la mano de IBM. Aunque a día de hoy sigue siendo funcional presenta ciertas limitaciones que actualmente te empujan con fuerza a GPT:

• CANTIDAD MÁXIMA DE PARTICIONES: 4 primarias. Si se necesitan más particiones, se debe convertir una de las particiones primarias a extendida. No obstante dentro de la partición extendida, se admiten entre 15 y 23 particiones lógicas.

• TAMAÑO MÁXIMO DE LAS PARTICIONES: 2TB Aunque puede parecer una cantidad de espacio grande, actualmente en entornos productivos empresariales podría no ser suficiente.

☣ FSTAB

Utiliza blkid para imprimir el UUID {Identificador Único Universal} de un dispositivo. Puede utilizarse con UUID= como una forma más robusta de nombrar dispositivos que funciona incluso si se añaden y eliminan discos.

☣  sudo blkid /dev/nvme2n1p1

/dev/nvme2n1p1 UUID="a57b0e02-1abb-4747-a13e-44c26d352bab" BLOCK_SIZE="4096" TYPE="ext4" PARTUUID="e864414c-5a84-9041-8242-390e8bdb9031" 


Una vez identificado el UUID de la partición, se añade al fichero /etc/fstab para que se monte automáticamente tras reiniciar el Sistema Operativo.

☣  sudo vim /etc/fstab

# /etc/fstab: static file system information.
#
UUID=750D-7A59 /boot/efi vfat umask=0077 0 2

UUID=293caccc-d9f3-4187-8d1b-f245b57713aa / xfs defaults 0 0

# PARTICIÓN MBR 
UUID="a57b0e02-1abb-4747-a13e-44c26d352bab" /mnt/MBRext4 ext4 defaults 0 0 




☣ GPT {GUID PARTITION TABLE}

Se trata de una tabla de particiones, sucesor del antiguo MBR, que admite dispositivos de almacenamiento de +2TB. Además permite crear tantas particiones como sea necesario {Windows admite hasta 128 particiones GPT}

El proceso de particionado es similar al de MBR, pero haciendo uso de la herramienta gdisk. En este caso trabajaré sobre el disco /dev/sdc
